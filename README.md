# NestFlow — Family Chief of Staff

NestFlow is a multilingual voice-activated household manager dashboard built for the Bolna x Cartesia Voc-A-Thon. It allows families to log groceries, assign chores, record domestic help instructions, and set reminders (medication, birthdays, festivals, travel prep) via simple phone calls or voice chats, instantly rendering update triggers live on a digital glassmorphic dashboard.

---

## Problem Statement

**Who & when:** Busy Indian household members (and their parents) call NestFlow hands-free — while cooking, commuting, or mid-chore — instead of typing into an app.

**What each call accomplishes:** The agent captures a household action in the caller's own language — add a grocery item, assign a chore, leave an instruction for domestic help (cook/maid/driver), or set a reminder (medication, birthday, festival, travel prep) — or reads back what is currently pending.

**Outcomes:**
- **Yes / clear** → the agent calls the matching tool, confirms aloud, and the item appears live on the dashboard.
- **No / cancelled** → nothing is written; the agent confirms the list is unchanged.
- **Unclear** → the agent asks one clarifying question (e.g. quantity, who it's for) before writing, rather than guessing.

**Target metric:** Household-coordination items successfully captured per call without follow-up correction.
*Baseline:* shared task lists/whiteboards capture roughly **60%** of intended items (the rest are forgotten or never written down).
*Goal:* **≥ 90%** capture rate via voice, with a realistic 30-day target of cutting "forgotten errand/instruction" incidents in half.

> **Why voice:** the moments these tasks occur (hands wet, driving, talking to the cook) are exactly when typing fails — and serving non-English-first family members requires speech, not forms.

## Tech Stack
- **Voice AI Orchestration:** Bolna AI
- **Voice Synthesis:** Cartesia (sonic-3-preview)
- **Local Server:** Node.js, Express, Socket.io
- **Database:** Local persistent JSON database (`db.json`)
- **Frontend CSS:** Glassmorphism UI using Custom Vanilla CSS variables

## Getting Started

### 1. Installation
Clone/navigate to this repository and install the dependencies:
```bash
npm install
```

### 2. Start the Local Server
Launch the server on port `3000`:
```bash
npm start
```
The server will boot and serve the frontend dashboard at `http://localhost:3000`.

> **Optional configuration:** copy `.env.example` to `.env` to override the
> `PORT`, or set `API_TOKEN` to require a shared secret on the Bolna-facing
> endpoints (`/api/groceries`, `/api/chores`, `/api/instructions`,
> `/api/summary`, `/api/webhook/call-ended`). When set, send it as the
> `Authorization` header (Bolna's `api_token` field). Left unset, the server
> stays open for zero-config local demos.

### 3. Expose Your Server to the Internet
Since Bolna runs in the cloud, it needs a public URL to send HTTP requests to your local server.
Install and run `ngrok` (or a similar tool) to forward traffic:
```bash
ngrok http 3000
```
Copy the forwarding HTTPS URL generated by ngrok (e.g., `https://a1b2-34-56-78.ngrok-free.app`).

## Bolna AI Agent Configuration

> **Deployed agent (for evaluation):** the full exported Bolna agent config is committed as
> [`bolna-agent-config.json`](./bolna-agent-config.json) — Agent ID `66d5e600-cc0f-4587-b50f-35880cb7b7a1`.
> It includes the 5 custom tools, the 4-language system prompt (English/Hindi/Telugu/Kannada),
> the Cartesia voices, the knowledge base, and the post-call extraction (Custom Analysis) used
> by the dashboard's "Call Insights" panel. The ngrok URL inside it is an ephemeral tunnel
> snapshot and will differ from your own run.

Log in to [platform.bolna.ai](https://platform.bolna.ai) to configure your Voice Agent.

### 1. Voice & Speech Synthesis (Cartesia)
1. Navigate to **Integrations** in the Bolna dashboard and make sure **Cartesia** is connected (using your Cartesia API key or hackathon credits).
2. In the **Agent Builder** under Voice Synthesis, choose **Cartesia** as the TTS provider.
3. Select an expressive model (e.g., `sonic-huggingface` or `sonic-3-preview`) and pick a natural-sounding voice profile.

### 2. System Instructions (LLM Prompt)
Paste the following into your Agent System Prompt. This is the **primary English agent**
(persona: **Mira**, a warm 25-year-old household assistant). The opening line is in English.

> The agent can be extended to Hindi, Telugu, and Kannada by adding per-language spoken
> phrase banks; this README documents the English agent only.

```text
# IDENTITY
You are Mira, a warm, capable 25-year-old household assistant for NestFlow — the
family's voice-based "Chief of Staff." You sound like an organised, friendly elder-
sister: calm, upbeat, efficient, never chatty. Keep every turn short and natural —
this is a phone call.

# LANGUAGE
Speak English. Open every call in English with the opening line below. If the caller
uses an occasional non-English word, understand it but keep replying in English.

# OBJECTIVE
Capture household actions and read back what is pending, using your tools:
1. add_grocery — item (required), quantity, category (Produce/Dairy/Pantry/Bakery/Household/Other).
2. assign_chore — title (required), assignee, priority (low/medium/high), notes.
3. log_instruction — instruction (required), recipient (Cook/Maid/Driver/Gardener), priority.
4. add_reminder — title (required), date, type (Medication/Birthday/Event/Festival/Travel/Other), notes.
5. get_summary — when the caller asks what's pending; read groceries, chores, instructions, and reminders back clearly.
Confirm every action in one short sentence, then ask if there is anything else they need.

# CONVERSATION PATHS
- Clear request → call the matching tool, then confirm using the corresponding phrase (which ends by asking if there is anything else).
- Missing/ambiguous detail → ask ONE clarifying question (quantity, or who it's for), then act.
- Caller cancels ("no, leave it") → do not call any tool; confirm nothing was added.
- Out of scope (orders, payments, bookings, weather, jokes) → decline in one line, steer back.
- Didn't catch it → ask them to repeat ONCE; don't loop.

# GUARDRAILS
- Never say you are an AI, a bot, or a language model. You are simply Mira.
- Never read out JSON, tool names, or any technical detail.
- Never invent items the caller did not mention.

# EXIT CONDITION
When the caller is done (indicates they are finished, e.g. says "No, that's all" or "I'm done"), close warmly using the Closing phrase, then end the call. Do not ask "Anything else?" in this final turn.

# SPOKEN PHRASES
- Opening: "Hi, I'm Mira, your family's NestFlow assistant. How can I help you today?"
- Fillers: "Got it!" / "Sure!" / "One moment." / "Let me check."
- Added grocery: "Done — I've added {item} to your grocery list. Anything else?"
- Assigned chore: "Done — I've assigned {chore} to {name}. Anything else?"
- Logged instruction: "Okay — I've told the {recipient} to {instruction}. Anything else?"
- Added reminder: "Done — I'll remember {title}. Anything else?"
- Ask quantity: "How much should I add?"
- Ask who for: "Who is this for?"
- Didn't catch: "Sorry, I didn't catch that — could you say it again?"
- Out of scope: "I can't help with that, but I can manage your groceries, chores, and
  help instructions. What would you like to add?"
- Closing: "Great, take care!"
```

### 3. Register Custom Tools in Bolna

In your Agent's **Tools Tab**, add one custom function per tool. The five ready-to-paste
packets live in [`bolna-tools.md`](./bolna-tools.md) — that file is the single source of
truth for the tool definitions. Paste each **complete** packet (not just the `parameters`
block), then replace `https://YOUR-NGROK-SUBDOMAIN.ngrok-free.app` with your live ngrok URL.

> **Important — Bolna's format is not plain OpenAI JSON.** Each tool is one object with
> top-level `name`, `description`, `parameters`, `key` (must be `"custom_task"`), and a
> `value` block holding `method`, `url`, and `param`. Pasting only the inner `parameters`
> schema causes the error *"Function name must be a string."*

The five tools (endpoints served by this backend):

| Tool name | Method | Endpoint | Purpose |
|---|---|---|---|
| `add_grocery` | POST | `/api/groceries` | Add a grocery / shopping item |
| `assign_chore` | POST | `/api/chores` | Assign a chore to a family member |
| `log_instruction` | POST | `/api/instructions` | Leave an instruction for domestic help |
| `add_reminder` | POST | `/api/reminders` | Add a reminder (medication, birthday, event, festival, travel) |
| `get_summary` | GET | `/api/summary` | Read back pending groceries, chores, instructions, reminders |

Example — the complete packet for `add_grocery` (the other three follow the same shape in
`bolna-tools.md`; `get_summary` uses `"method": "GET"` with an empty `param`):

```json
{
  "name": "add_grocery",
  "description": "Call when the user wants to add a grocery or shopping item to the household shopping list. Capture the item name, and the quantity and category if mentioned.",
  "pre_call_message": "Got it, adding that to the grocery list.",
  "parameters": {
    "type": "object",
    "properties": {
      "item": { "type": "string", "description": "Name of the grocery item, e.g. 'Milk', 'Tomatoes', 'Atta'" },
      "quantity": { "type": "string", "description": "Amount to buy with units, e.g. '2 litres', '1 kg'. Use 'unspecified' if not mentioned." },
      "category": { "type": "string", "enum": ["Produce", "Dairy", "Pantry", "Bakery", "Household", "Other"], "description": "Grocery category" }
    },
    "required": ["item"]
  },
  "key": "custom_task",
  "value": {
    "method": "POST",
    "param": { "item": "%(item)s", "quantity": "%(quantity)s", "category": "%(category)s" },
    "url": "https://YOUR-NGROK-SUBDOMAIN.ngrok-free.app/api/groceries",
    "api_token": "",
    "headers": {}
  }
}
```

> If you set `API_TOKEN` on the server, put `"Bearer your-token-value"` in each tool's
> `api_token` field (the server strips the `Bearer ` prefix).
>
> *Optional:* the backend also exposes `POST /api/groceries/bought` (mark an item as bought
> by `item` name or `id`) if you want to add a fifth "mark as bought" tool later.

### 4. Configure Call Completed Webhook
To sync call transcripts, durations, and logs to the dashboard bottom panel:
1. In the Bolna dashboard, go to the **Analytics / Webhook Settings** tab.
2. Select event type: `completed` (or "Call Completed").
3. Set the webhook URL to: `<YOUR_NGROK_URL>/api/webhook/call-ended`.
4. Save the webhook.

### 5. Configure Custom Analysis (Extraction) — *scored feature*
The dashboard renders a **"Call Insights"** panel from Bolna's post-call extraction,
which demonstrates the *custom analysis / variables* evaluation criterion live on screen.
1. In the Bolna agent's **Analytics → Custom Analysis / Extraction** tab, define an
   extraction schema. Recommended fields for NestFlow:
   - `detected_language` — the language the caller actually used (showcases multilinguality)
   - `caller_intent` — e.g. `add_grocery`, `assign_chore`, `leave_instruction`, `query_list`
   - `items_captured` — what was added or changed during the call
   - `unresolved` — anything the agent could not complete
2. Bolna includes these as `extracted_data` in the call-completed webhook. The backend
   captures the object automatically — no schema change needed — and shows each field as a
   labelled chip. Any keys you define will appear; the panel is hidden when extraction is empty.

## Verifying the Flow Locally

You can simulate Bolna custom tool calls without phone lines to verify the real-time websocket pushes to your dashboard:

**Simulate adding a grocery item:**
```bash
curl -X POST http://localhost:3000/api/groceries \
  -H "Content-Type: application/json" \
  -d '{"item": "Apples", "quantity": "6 units", "category": "Produce"}'
```

**Simulate assigning a chore:**
```bash
curl -X POST http://localhost:3000/api/chores \
  -H "Content-Type: application/json" \
  -d '{"title": "Water the terrace plants", "assignee": "Achyut", "priority": "medium"}'
```

**Simulate a call completion webhook (with custom-analysis insights):**
```bash
curl -X POST http://localhost:3000/api/webhook/call-ended \
  -H "Content-Type: application/json" \
  -d '{"id": "call-12345", "transcript": "User: Milk add kar do. Agent: Theek hai, maine milk grocery list me add kar diya hai.", "conversation_duration": 15, "extracted_data": {"detected_language": "Hinglish", "caller_intent": "add_grocery", "items_captured": "Milk"}}'
```

When you trigger these, you will see elements fly onto your dashboard immediately with custom toasts sliding in!
